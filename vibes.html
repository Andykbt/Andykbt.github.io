vibes

Project Description

vibes is a web application that gets music recommendations based on the weather of a city. Sometimes the weather is a good indicator of your current mood. This app lets users search for cities in the world, get it's weather data and then get song recommendations based on the weather.

Built using React, Node JS, and Express, it uses weather descriptors such as "Cloudy", "Sunny", etc. to determine the current "mood".

What I Learnt

This was my first time using Node JS in a way that didn't involve anything relatively simple. I had to learn how to create a server and a client that had to regularly communicate with each other.
I've also learned the authentication process using Spotify, where a user would click "login" to get redirected to another page where they would authorise access to their data, then it would get redirected back to the site with the user data. In this app, this is done by passing the user's access token that is fetched from the authentication process and used in the hash parameters of the website link. This is my why when logging in with Spotify, you may see a long hash.

On the frontend side, I used React as I don't believe that I've mastered it and want to work on my React skills more. One of the main struggles I had was making the web app look better through animations. I've only ever done really basic animations in my previous projects so wanted to test myself and try to make more complex animations. This was another one of the problems that I had to overcome and I definitely spent a couple hours headscratching on getting the right animations to work.
